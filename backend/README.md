# Mindstash Backend API Documentation

## Base URL
http://localhost:3000/api/v1

## Authentication
Most endpoints require authentication via a JWT token. Include the token directly in the `Authorization` header of your request:
Authorization: <your_jwt_token>

## Response Format
Responses return standard JSON objects. Successful requests typically return the requested data or a success message, while errors will return a `message` field describing the issue.

```json
{
  "message": "Response message string",
  "content": [ ... ] 
}
```

# 1. Authentication APIs

## 1.1 Sign Up
POST /signup

Access: Public

Description: Register a new user account with a username and password.

Request Body:

```json
{
  "username": "johndoe",
  "password": "securepassword123"
}
```

Response (Success):

```json
{
  "message": "User signed up"
}
```

Response (Error - User exists):

```json
{
  "message": "User already exists"
}
```

## 1.2 Sign In
POST /signin

Access: Public

Description: Authenticate an existing user to receive a JWT token.

Request Body:

```json
{
  "username": "johndoe",
  "password": "securepassword123"
}
```

Response (Success):

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

Response (Error - Wrong Credentials):

```json
{
  "message": "Incorrrect credentials"
}
```

# 2. Content Management APIs
Access: Authenticated users (Requires Authorization header)

## 2.1 Add Content
POST /content

Description: Save a new piece of content (link, tweet, video, etc.) to the user's stash.

Request Body:

```json
{
  "title": "My Favorite Article",
  "link": "[https://example.com/article](https://example.com/article)",
  "type": "article"
}
```

Response:

```json
{
  "message": "Content added"
}
```

## 2.2 Get User Content
GET /content

Description: Retrieve all content saved by the currently authenticated user. The user's username is also populated in the response.

Response:

```json
{
  "content": [
    {
      "_id": "65b123456789abcde1234567",
      "title": "My Favorite Article",
      "link": "[https://example.com/article](https://example.com/article)",
      "type": "article",
      "tags": [],
      "userId": {
        "_id": "65b123456789abcde1234560",
        "username": "johndoe"
      },
      "__v": 0
    }
  ]
}
```

## 2.3 Delete Content
DELETE /content

Description: Delete a specific piece of content from the user's stash using its ID.

Request Body:

```json
{
  "contentId": "65b123456789abcde1234567"
}
```

Response:

```json
{
  "message": "Deleted"
}
```

# 3. Brain Sharing APIs

## 3.1 Toggle Share Link
POST /brain/share

Access: Authenticated users

Description: Generate a public shareable link for the user's entire stash, or revoke an existing link.

Request Body (To Enable Sharing):

```json
{
  "share": true
}
```

Response (Returns existing or new hash):

```json
{
  "hash": "aB3dE8fGhI"
}
```

Request Body (To Disable Sharing):

```json
{
  "share": false
}
```

Response:

```json
{
  "message": "Removed link"
}
```

## 3.2 Get Shared Brain (Public)
GET /brain/:shareLink

Access: Public

Description: Retrieve the contents of a user's stash using their public share hash.

Path Parameter:

shareLink: The 10-character hash generated by the /brain/share endpoint.

Response:

```json
{
  "username": "johndoe",
  "content": [
    {
      "_id": "65b123456789abcde1234567",
      "title": "My Favorite Article",
      "link": "[https://example.com/article](https://example.com/article)",
      "type": "article",
      "tags": [],
      "userId": "65b123456789abcde1234560",
      "__v": 0
    }
  ]
}
```

Response (Error - Invalid Hash):

```json
{
  "message": "Sorry incorrect input"
}
```

# 4. Error Responses

All API endpoints return appropriate HTTP status codes combined with JSON messages:

403 Forbidden

```json
{
  "message": "You are not logged in"
}
```

411 Length Required / Invalid State
Used when data conflicts exist (e.g., user already exists) or a requested resource is invalid.

```json
{
  "message": "User already exists"
}
```

500 Internal Server Error

```json
{
  "message": "Something went wrong while deleting content",
  "error": "Detailed error string"
}
```

# 5. Environment Variables

Create a .env file in the root of the backend directory with the following variable:

```env
MONGODB_URI="mongodb+srv://username:password@cluster.mongodb.net/mindstash"
```

(Note: The JWT password is currently hardcoded in src/config.ts as !23123. For production, it is highly recommended to move this to an environment variable as well).

# 6. Frontend Integration Examples

## Sign In Flow

```javascript
const login = async (username, password) => {
  const response = await fetch('http://localhost:3000/api/v1/signin', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ username, password })
  });

  const data = await response.json();

  if (data.token) {
    localStorage.setItem('token', data.token);
  } else {
    alert(data.message);
  }

  return data;
};
```

## Authenticated Request (Fetch Content)

```javascript
const getUserContent = async () => {
  const token = localStorage.getItem('token');

  const response = await fetch('http://localhost:3000/api/v1/content', {
    method: 'GET',
    headers: {
      'Authorization': token, // Middleware expects raw token
      'Content-Type': 'application/json'
    }
  });

  const data = await response.json();
  return data.content;
};
```

## Generate Share Link

```javascript
const toggleShare = async (isSharing) => {
  const token = localStorage.getItem('token');

  const response = await fetch('http://localhost:3000/api/v1/brain/share', {
    method: 'POST',
    headers: {
      'Authorization': token,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ share: isSharing })
  });

  const data = await response.json();

  if (data.hash) {
     return `http://your-frontend-url/share/${data.hash}`;
  }
  return data.message;
};
```

# 7. Best Practices

## Token Management

Pass the exact JWT token into the Authorization header without a "Bearer " prefix, as req.headers['authorization'] expects the string directly in the current middleware implementation.

Store tokens securely on the frontend (e.g., in localStorage or HttpOnly cookies).

## Data Types

Ensure the type field passed to the content creation endpoint aligns with your frontend icons/renders (e.g., 'youtube', 'twitter', 'article').

The tags array expects Mongoose ObjectIds. If implementing a tag feature later, ensure tags are created in the database first before linking them.
